# Runtime Renderer 设计文档

> 状态：已实现
> 前置：Runtime Auto-Load alpha (M5) 已完成
> 目标：让目标 Agent 在启动时能原生消费 AgentMesh registry 中的 skill

## 1. 核心问题

Alpha 实现生成的 LoadPlan JSON 和 bootstrap shim **不被任何目标 Agent 原生消费**。
shim 写入的 `agentmesh_loader.py` 不会被 Hermes/OpenClaw/Cursor/Windsurf 自动执行。

## 2. 解决方案：RuntimeRenderer

每个 Agent 有不同的原生格式。渲染器将 registry skills 转换为目标 Agent 原生理解的文件格式。

### 渲染策略矩阵

| Target | 原生格式 | 渲染策略 | 文件位置 |
|---|---|---|---|
| hermes | SKILL.md (Markdown) | 合并 allowed skills 为一个 skill | `~/.hermes/skills/custom/agentmesh-loader/SKILL.md` |
| openclaw | SKILL.md (Markdown) | 同 hermes | `~/.openclaw/workspace/skills/agentmesh-loader/SKILL.md` |
| cursor | .mdc (Markdown + YAML frontmatter) | 渲染为规则文件 | `~/.cursor/rules/agentmesh-rules.mdc` |
| windsurf | .md (Markdown) | 渲染为规则文件 | `~/.windsurf/rules/agentmesh-rules.md` |
| codex | SKILL.md | Compatibility Sync（不走渲染器） | 由 sync 流程处理 |
| claude-code | — | export-only（不走渲染器） | 由 export 流程处理 |
| aider | .md (Markdown) | 渲染为 conventions 文件 | `~/.aider.conventions.md` |

### Renderer 接口

```python
class RuntimeRenderer(ABC):
    """Render registry skills into target Agent native format."""
    
    target: str
    
    @abstractmethod
    def render(self, home: Path, allowed_skills: list[str], metadata: dict) -> list[FilePayload]:
        """Render allowed skills into native files.
        
        Args:
            home: AgentMesh home/registry path
            allowed_skills: list of skill names from LoadPlan
            metadata: {plan_id, generated_at, target, registry}
        
        Returns:
            list of FilePayload to write
        """
    
    @abstractmethod  
    def entrypoint_path(self, skill_dir: Path) -> Path:
        """Return the primary file path for this renderer."""
```

### FilePayload

```python
@dataclass
class FilePayload:
    path: Path          # 绝对路径
    content: str        # 文件内容
    overwrite: bool     # 是否允许覆盖
    description: str    # 文件描述（用于 dry-run 展示）
```

## 3. 实现计划

### P1: RuntimeRenderer 基类 + Hermes/OpenClaw 渲染器

将 allowed skills 的 SKILL.md 内容合并渲染到一个 `agentmesh-loader/SKILL.md` 中。

```markdown
# AgentMesh Auto-Loaded Skills

> Generated by AgentMesh at {timestamp}
> Plan ID: {plan_id}
> Registry: {registry}
> Allowed: {n} skills | Blocked: {m} skills

---

{每个 allowed skill 的 SKILL.md 内容，用 hr 分隔}

---

*Managed by AgentMesh Runtime Auto-Load. Do not edit manually.*
*Use `agentmesh runtime disable --target {target}` to remove.*
```

### P2: Cursor/Windsurf 规则文件渲染器

将 skills 渲染为 Cursor 的 `.mdc` 格式：

```markdown
---
description: "AgentMesh auto-loaded skills ({n} rules)"
alwaysApply: true
---

# AgentMesh Skills

{每个 skill 的关键内容}
```

### P3: 集成到 bootstrap 流程（已实现）

修改 `apply_bootstrap()`：
1. 生成 LoadPlan
2. 调用对应 Renderer 渲染文件
3. 写入渲染后的文件（而非空模板）
4. 记录审计日志

### P4: 审计日志（已实现）

每次 bootstrap/disable 记录到 `<home>/state/runtime-audit/`。

## 4. 安全约束

- 渲染器只输出 **声明式内容**（Markdown/规则文本），不生成可执行代码
- `agentmesh_loader.py` entrypoint 不再是主要交付物
- 不覆盖非 managed 文件（保持 unmanaged 检测）
- secrets 脱敏后才能进入渲染内容
- blocked skills 完全排除，不出现在渲染输出中
