## 🚀 快速开始

### 基础使用

```bash
# 1. 启动高级服务器
am local serve --advanced --host 127.0.0.1 --port 9090

# 2. 访问Web界面
浏览器打开: http://127.0.0.1:9090
```

### 第一次使用

1. **界面概览**：左侧为命令导航面板，右侧为工作区
2. **系统状态**：顶部显示连接状态和版本信息
3. **核心功能**：快速命令 + 高级功能分类

## 🔧 功能详解

### 1. 批量操作 📋

#### 使用场景
- **多Agent同步**：一次对多个Agent执行相同操作
- **批量skills管理**：同时扫描、导入多个技能包
- **系统维护**：批量执行诊断、审计等系统操作

#### 操作流程
1. **选择命令**：勾选需要执行的命令
2. **设置目标**：（可选）输入目标Agents，如 `hermes,claude_code`
3. **生成计划**：点击"📋 生成计划"预览执行内容
4. **执行批量**：点击"🚀 执行批量"运行所有选定命令

#### 技术特性
- **异步执行**：支持并发命令执行
- **执行摘要**：显示成功/失败统计和成功率
- **错误处理**：单个命令失败不影响其他命令执行
- **计划保存**：支持保存批量计划供后续参考

### 2. 结果导出 📤

#### 导出格式

| 格式 | 用途 | 特点 |
|------|------|----------|
| **JSON** | 数据备份 | 结构化、完整、支持程序处理 |
| **CSV** | 数据分析 | Excel兼容、表格化、易于统计 |
| **TXT** | 文本查看 | 纯文本、日志格式、便于阅读 |

#### 导出类型

1. **命令历史导出**：
   - 可设置记录数量限制（1-1000条）
   - 包含执行时间、状态、输出内容
   - 自动添加时间戳到文件名

2. **批量结果导出**：
   - 按批量任务ID导出执行结果
   - 包含详细的成功/失败统计
   - 支持失败详情查看

#### 导出操作
```bash
# 通过Web界面导出（推荐）
# 点击左侧"结果导出" → 选择格式和数量 → 点击"导出历史"

# 通过命令行导出（高级）
am local serve --advanced
# 然后使用 curl 或类似工具访问导出API
```

### 3. 定时任务 ⏰

#### Cron表达式

AgentMesh 使用标准Cron表达式格式：

```
┌───── 分钟 (0-59)
│ ┌─── 小时 (0-23)
│ │ ┌───── 日期 (1-31)
│ │ │ ┌───────── 月份 (1-12)
│ │ │ │ ┌─────────── 星期 (0-6)
* * * * * * * * * * * *
```

#### 常用示例

| Cron表达式 | 说明 | 示例 |
|------------|------|----------|
| `0 9 * * *` | 每天上午9点执行 | 每日技能扫描 |
| `0 */4 * * *` | 每4小时执行一次 | 定期系统检查 |
| `0 0 * * 1-5` | 周一到周五执行 | 工作日自动化 |
| `*/30 * * * *` | 每30分钟执行 | 高频监控任务 |
| `0 9,12,18 * * *` | 每天9点、12点、18点执行 | 多次日常任务 |

#### 任务管理

1. **创建任务**：选择命令、输入Cron表达式、设置名称
2. **启用/禁用**：随时启用或暂停定时任务
3. **查看详情**：显示执行历史和运行次数
4. **删除任务**：移除不再需要的定时任务

#### 使用场景

- **定期维护**：每天扫描skills，检查系统健康
- **数据同步**：定时同步memory和models
- **自动化报告**：定期运行审计生成报告
- **备份任务**：设定定期备份registry

### 4. 收藏夹 ⭐

#### 收藏功能

1. **添加收藏**：在命令执行界面添加常用命令到收藏
2. **分类管理**：收藏夹按命令类型分组
3. **快速执行**：直接从收藏夹执行命令
4. **收藏管理**：删除不再需要的收藏命令

#### 最佳实践

- **按使用频率分类**：高频命令放前面便于访问
- **描述性命名**：为收藏添加描述说明使用场景
- **定期整理**：删除不常用的收藏，保持收藏夹整洁
- **批量操作结合**：收藏的命令可以用于批量执行

### 5. 执行历史 📜

#### 历史记录

- **完整记录**：所有命令执行都会记录，包括成功和失败
- **详细信息**：记录命令内容、执行时间、耗时、输出结果
- **状态标识**：✅ 成功、❌ 失败、⚠️ 警告
- **自动编号**：每次执行自动分配唯一ID

#### 历史操作

1. **查看历史**：按时间倒序查看所有执行记录
2. **筛选显示**：支持按状态筛选历史记录
3. **清空显示**：清空当前列表显示（不影响存储）
4. **历史导出**：将历史记录导出为文件

#### 历史用途

- **问题排查**：查看失败命令的错误信息
- **性能分析**：分析命令执行时间，发现性能问题
- **操作审计**：追踪谁在何时执行了什么命令
- **回溯分析**：查看操作历史，理解问题发生过程

## 💡 最佳实践

### 工作流优化

1. **dry-run优先**：
   - 始终先用`--dry-run`预览操作
   - 确认无误后再执行`--apply`
   - 避免意外修改系统

2. **批量操作策略**：
   - 相关命令一起批量执行
   - 分批处理大量操作
   - 每批操作后验证结果

3. **定时任务设置**：
   - 避免高峰时段设置重要任务
   - 设置合理的执行频率
   - 为长时间任务添加超时机制

4. **数据管理**：
   - 定期导出历史记录备份
   - 清理过期的历史数据
   - 定期整理收藏夹

### 安全考虑

1. **命令验证**：
   - 查看命令内容后再执行
   - 避免执行破坏性命令
   - 使用dry-run模式验证复杂命令

2. **访问控制**：
   - 服务器默认只绑定127.0.0.1
   - 不要将端口暴露到公网
   - 在防火墙中限制访问

3. **数据安全**：
   - 导出的文件及时删除敏感信息
   - 定期更换端口和密钥
   - 避免在终端显示完整命令参数

### 性能优化

1. **界面响应**：
   - 减少同时打开的功能页面
   - 合理设置历史记录数量限制
   - 定期清理浏览器缓存

2. **命令执行**：
   - 批量操作选择合理批次大小
   - 避免单个命令执行时间过长
   - 利用定时任务减少手动操作

## 🔍 故障排除

### 常见问题

#### 连接问题

**问题**：无法访问Web界面
```
✅ 解决方案：
1. 检查服务器是否启动：`am local serve --advanced`
2. 检查端口是否正确：默认9090
3. 检查防火墙是否阻止访问
4. 尝试使用其他端口：`am local serve --advanced --port 9091`
```

#### 功能异常

**问题**：点击按钮无响应
```
✅ 解决方案：
1. 查看浏览器控制台是否有JavaScript错误
2. 检查网络连接状态
3. 刷新页面重试
4. 清除浏览器缓存后重试
```

#### 命令执行失败

**问题**：命令执行一直显示"执行中"
```
✅ 解决方案：
1. 检查命令语法是否正确
2. 查看历史记录中的错误信息
3. 尝试单独执行命令找出问题
4. 检查Agent runtime是否正常运行
```

#### 滚动位置丢失

**问题**：切换功能时右侧区域跳到顶部
```
✅ 解决方案：
1. 这是已修复的问题，新版本自动保持滚动位置
2. 如仍遇到，手动滚动回原来的位置
```

### 日志和调试

1. **启用详细日志**：
   ```bash
   # 启动服务器时添加日志输出
   am local serve --advanced --verbose
   ```

2. **浏览器调试**：
   - 打开浏览器开发者工具（F12）
   - 查看Console标签的错误信息
   - 检查Network标签的API请求状态

3. **常见错误码**：
   - `500`：服务器内部错误
   - `501`：不支持的HTTP方法
   - `504`：请求超时
   - `422`：请求参数验证失败
